# Benchmarks de Productividad — GitHub Copilot

**Fuente principal:** GitHub, McKinsey, Accenture (estudios 2024–2025)
**Relevancia:** Argumentar el valor de la adopción ante directivos de GCO

---

## Datos clave de GitHub (estudio oficial, 2024)

- **55% más rápido** en tareas de codificación (según developers que participaron en el estudio controlado de GitHub)
- **46% de código generado** o sugerido por Copilot en proyectos activos de empresas que lo adoptaron
- **74% de usuarios** reportaron sentirse más enfocados en trabajo satisfactorio (menos tareas repetitivas)
- **88% de usuarios** reportaron mayor productividad

## Accenture (2024) — Despliegue en 50.000 empleados

- Accenture desplegó GitHub Copilot a 50.000 desarrolladores internos
- Resultado: reducción del 35% en tiempo de revisión de código
- Identificaron que el mayor valor apareció en **comprensión de código legacy**, no en generación de código nuevo — relevante para el stack COBOL/legacy de GCO

## McKinsey (2023) — "The economic potential of generative AI"

- Los desarrolladores que usan asistentes de IA completan tareas hasta un **40% más rápido**
- El mayor impacto se da en: documentar código, entender código heredado, generar tests y preparar primeras versiones

---

## Casos relevantes para COBOL y legacy

- **BMO (Bank of Montreal):** Implementó GitHub Copilot para equipos con COBOL y mainframe. Resultado: reducción de carga en tareas de análisis de impacto y documentación.
- **FedEx:** Uso de IA para entender código legacy. El caso de uso principal fue el análisis de código existente antes de modificaciones.
- **Pattern general de adopción legacy:** El primer valor no aparece en generación de código nuevo, sino en comprensión de código existente — análisis de logs, comprensión de bloques COBOL, revisión de impacto en Copybooks.

## Cómo usar estos datos en el proyecto

### Para la presentación a directivos
> *"Según el estudio de GitHub, developers que usan Copilot completan tareas de codificación un 55% más rápido. En GCO, esto tiene impacto directo en tareas con COBOL y DB2, donde analizar impacto o entender código heredado puede pasar de horas a minutos."*

### Para posts en Teams (con filtro GCO)
> *"Un developer que usó Copilot para analizar una consulta DB2 compleja pasó de 2 horas a 20 minutos. No por magia — por dar el contexto correcto."*

### Para responder la objeción "¿vale para COBOL?"
> *"Accenture desplegó Copilot en 50.000 developers, incluidos equipos con código legacy. El mayor impacto lo encontraron exactamente ahí: comprensión de código heredado y análisis de impacto."*
