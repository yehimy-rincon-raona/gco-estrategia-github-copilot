# Video 2: Primeros pasos

## Storytelling y guion estructurado

El objetivo de este segundo vídeo es dar el primer paso real: abrir GitHub Copilot Chat en Visual Studio y hacer la primera interacción. Sin configuraciones avanzadas, sin modo agente todavía — solo la conversación más básica para que el developer vea que Copilot responde y que ya está disponible en su entorno habitual.

### Título del vídeo

**Primeros pasos: tu primera conversación con Copilot en Visual Studio**

### Concepto narrativo

Con el entorno listo, el siguiente paso es tan simple como abrir el panel de chat y escribir. Este vídeo muestra dónde está Copilot dentro de Visual Studio, cómo se abre el chat, cómo funciona el completado en línea y cómo se hace la primera pregunta sobre código — sin salir del IDE, sin buscar en el navegador.

## Estructura del guion y guía de producción

---

### 1. Conexión con el vídeo anterior · 0:00 – 0:10

> "El entorno ya está listo: Visual Studio instalado, cuenta corporativa validada, Copilot activo.
>
> Ahora el siguiente paso: abrir el chat y hablar con él por primera vez."

**Fuente:** Continuidad narrativa del Vídeo 1

**Visuales**
Pantalla de Visual Studio con el proyecto de prueba abierto. Texto en pantalla: **"Entorno listo → Primer uso"**.

### 2. Localizar y abrir Copilot Chat · 0:10 – 0:30

> "Copilot Chat está integrado directamente en Visual Studio. Para abrirlo, vamos al menú Ver, luego a Otros Windows, y seleccionamos GitHub Copilot Chat.
>
> El panel aparece en la parte derecha del IDE, sin abrir ninguna herramienta externa."

**Fuente:** Sesión 1 G5M · **Minuto ref:** ~31:48

**Visuales**
Mostrar el recorrido: menú **Ver → Otros Windows → GitHub Copilot Chat**. Hacer zoom a cada paso. Panel abierto a la derecha. Texto en pantalla: **"Ver → Otros Windows → GitHub Copilot Chat"**.

---

### 3. Lo que ves al abrirlo · 0:30 – 0:50

> "El panel muestra un campo de texto donde escribir y, arriba, el contexto activo: qué archivo tiene Copilot en cuenta en este momento.
>
> Esto es importante: Copilot no trabaja de forma genérica. Responde en función del código que tienes abierto en el editor."

**Fuente:** Sesión 1 G4M / Sesión 1 G5M · **Minuto ref:** ~30:38

**Visuales**
Zoom al campo de texto del chat. Zoom al indicador de contexto activo (nombre del archivo abierto). Texto en pantalla: **"Copilot ve lo que tienes abierto"**.

---

### 4. La primera pregunta · 0:50 – 1:10

> "Para la primera interacción, usamos algo concreto y directo: seleccionamos un fragmento de código, escribimos #selection en el chat y le pedimos a Copilot que lo explique.
>
> No hace falta un prompt elaborado. Con esto ya tenemos la primera respuesta útil dentro del IDE, sin salir de Visual Studio."

**Fuente:** Sesión 1 G4M · Ejemplo: explicación de código en COBOL o .NET

**Visuales**
Mostrar selección de código en el editor. Escribir en el chat: `Explica este código #selection`. Zoom a la respuesta generada. Texto en pantalla: **"#selection → referencia lo que tienes seleccionado"**.

---

### 5. Sugerencias en línea mientras escribes · 1:10 – 1:28

> "Además del chat, Copilot también trabaja directamente en el editor mientras escribes. Cuando aparece texto en gris, esa es la sugerencia de Copilot.
>
> Si te sirve, presionas Tab para aceptarla. Si no, simplemente sigues escribiendo y la sugerencia desaparece."

**Fuente:** Sesión 1 G6M / Sesión 1 G5M

**Visuales**
Mostrar sugerencia inline apareciendo en gris en el editor. Mostrar tecla Tab para aceptar. Texto en pantalla: **"Tab para aceptar · Sigue escribiendo para ignorar"**.

---

### 6. Cierre y siguiente paso · 1:28 – 1:42

> "Con esto ya tienes los dos puntos de entrada básicos: el chat para preguntas y el completado en línea para sugerencias directas en el código.
>
> En el siguiente vídeo veremos cómo usar estos dos modos con casos reales del día a día."

**Fuente:** Estrategia evolutiva de adopción · Fase 2

**Visuales**
Resumen visual con los dos modos: **Chat + Completado inline**. Texto en pantalla: **"Siguiente: cómo interactuar con Copilot"**.

## Recomendaciones para edición rápida

- Mantener el vídeo entre **90 segundos y 100 segundos**.
- El recorrido Ver → Otros Windows → Copilot Chat debe verse claro — hacer zoom en cada clic.
- Difuminar cualquier texto de código real si pertenece a un proyecto de cliente.
- `#selection` debe verse escrito en el chat, no solo mencionado en el audio.
- Las sugerencias inline en gris son muy sutiles en pantalla — usar zoom o highlight de cursor para que se vean.
- No entrar todavía en modo agente ni en diferencias entre modos — eso es Vídeo 3.
